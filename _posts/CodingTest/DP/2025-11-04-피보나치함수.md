# 풀이

```c++
#include <bits/stdc++.h>

using namespace std;

pair<int, int> dp[41] = { {1, 0},{0, 1}, };

int main()
{
	ios_base::sync_with_stdio(false);
	cin.tie(NULL);
	cout.tie(NULL);
	
	int t;
	cin >> t;

	for (int i = 2; i < 41; i++)
		dp[i] = { dp[i - 1].first + dp[i - 2].first, dp[i - 1].second + dp[i - 2].second };

	for (int i = 0; i < t; i++)
	{
		int n;
		cin >> n;
		cout << dp[n].first << " " << dp[n].second << '\n';
	}

	return 0;
}
```
---------------------------------------

# 후기

피보나치 수열을 구하면서 나오는 0과 1을 카운팅해서 출력하는 문제.... 라고 생각했다면! 시간초과로 끝날것이다.

왜냐하면 n = 40이 되면 거의 int의 끝까지 가기때문에 수가 너무크다. 이 문제는 결국 뭐냐!

"Dynamic Programming" DP다.

dp[0] ~ dp[n의 최대치] 까지 처음에 구하고 시작할 수 있다. 왜냐면 0과 1은 1,0 0,1 이라는 값을 가지고있다.

피보나치처럼 F(n) = F(n - 1) + F(n - 2)라는 식이 성립한다. n - 1과 n - 2의 개수를 더한게 n의 0과 1의 개수다.

그렇게 메모이제이션한걸 바탕으로 테스트케이스가 들어오면 그 n에 값을 출력해주면 된다.
